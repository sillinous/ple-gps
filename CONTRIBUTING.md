## Our Philosophy

This repository operates on a few core principles:

**Ideas over authorship.** Contributions here become seeds for community growth, not monuments to individuals. If you submit an idea and it evolves into something bigger, celebrate that — it worked.

**Rigor over speed.** We're building frameworks meant to outlast current political cycles. We'd rather get it right than get it published.

**Accessibility over jargon.** Economic transformation affects everyone. Our writing and tools should be understandable by a 22-year-old factory worker and a 55-year-old policy researcher equally.

**Constructive dissent is healthy.** If you think something is wrong, say so — with evidence and alternatives. Disagreement, done well, makes the work stronger.

## Ways to Contribute

### 📝 Writing & Research
- Draft or expand position papers, explainers, or case studies
- Translate documents into other languages
- Review and fact-check existing content
- Synthesize research into the foundation document

### 🛠 Technical
- Build tools that help communities model economic transitions
- Improve the repo's infrastructure (CI, templates, automation)
- Create data visualizations of PLE concepts
- Contribute to platform development

### 🗣 Community & Facilitation
- Facilitate working group discussions
- Onboard new contributors
- Moderate community forums
- Organize local or virtual events

### 🎨 Design & Communication
- Improve accessibility of documentation
- Create infographics, visual explainers, or presentations
- Develop educational materials

### 💡 Ideas & Strategy
- Open discussion threads on underexplored topics
- Propose new working groups
- Identify gaps in the foundation document
- Challenge existing assumptions (constructively)

## Working Groups

The community is organized into Working Groups (WGs). Each WG owns a domain of the PLE framework:

| Working Group | Focus |
|---|---|
| **WG-THEORY** | Core economic models, UBI frameworks, post-scarcity theory |
| **WG-POLICY** | Transition policy, legislative frameworks, international models |
| **WG-TECH** | Automation mapping, AI impact analysis, platform tools |
| **WG-COMMUNITY** | Mutual aid networks, local implementation, grassroots organizing |
| **WG-COMMS** | Public messaging, media strategy, education |
| **WG-GOVERNANCE** | Community governance, DAO structures, democratic design |

To join a working group, tag it in your issue or PR, or introduce yourself in Discussions.

## Before You Start

1. **Check open issues** — your idea may already be in progress
2. **Open a Discussion** if you have a question or want feedback before investing time
3. **Introduce yourself** in the [Community Introductions](../../discussions/categories/introductions) thread
4. **Read the Foundation Document** (`/docs/foundation.md`) — it's the shared context everything builds from

For significant new directions, open a **Discussion** first. This prevents effort duplication and gets you early feedback.

## Submitting Work

### Branches
- Branch from `main`
- Name your branch descriptively: `wg-policy/ubi-transition-framework` or `docs/clarify-automation-taxonomy`

### Commits
```
[WG-TAG] Short description of change

Optional longer explanation.
Closes #123
```

Examples:
```
[WG-THEORY] Add section on post-scarcity resource allocation
[WG-COMMS] Improve introduction for non-academic audiences
[DOCS] Fix broken links in onboarding guide
```

### Pull Requests
- Use the PR template provided
- Link to the issue or discussion it addresses
- Tag the relevant Working Group(s)
- Be explicit about what changed and why

---

## Review Process

1. **Auto-checks** run on all PRs (spell check, link validation, formatting)
2. **WG Review** — at least one member of the relevant working group reviews
3. **Community window** — 48-hour window for broader community feedback on significant changes
4. **Merge** — maintainer or WG lead merges after approval

## Code of Conduct

We follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) with one addition: **economic precarity is a lived reality for many contributors here.** Be mindful that the stakes of this work are personal for many people. Compassion is not optional.
