# Governance

This document describes how decisions are made in the PLE/GPS project and who is responsible for what.

---

## Principles

1. **Subsidiarity** — decisions are made at the lowest appropriate level (working group → stewardship → community)
2. **Transparency** — reasoning behind significant decisions is documented publicly
3. **Meritocracy of ideas** — the quality of an argument matters more than who makes it
4. **Stewardship, not ownership** — the Foundation for Global Progress holds the project in trust for the community; it does not own the ideas

---

## Structure

### Working Groups
The primary unit of contribution and decision-making. Each working group:
- Has a defined domain (see `CONTRIBUTING.md`)
- Maintains its own discussion threads and documentation
- Can approve changes within its domain by consensus
- Nominates a **Lead** and a **Deputy Lead** (min. 3 months of active contribution required)

### Stewardship Team
A small group drawn from working group leads and Foundation staff. Responsibilities:
- Resolving cross-working-group disputes
- Approving governance changes
- Maintaining project integrity and CoC enforcement
- Onboarding new working groups

Current members: listed in [`community/STEWARDS.md`](STEWARDS.md)

### Community
All contributors, subscribers, and interested parties. Participates through:
- GitHub Discussions
- Working group meetings (open attendance)
- Periodic community calls

---

## Decision Types

| Type | Who Decides | Process |
|------|-------------|---------|
| Minor edits (typos, clarity) | Any contributor | PR + 1 approval |
| Content additions within a domain | Working group | PR + 1 WG member approval, 48hr window |
| Significant framework changes | Working group + community | Discussion thread, 7-day comment period, WG consensus |
| Cross-domain changes | Stewardship | Discussion thread, 14-day comment period |
| Governance changes | Stewardship + community | Discussion thread, 21-day comment period, recorded rationale |
| CoC enforcement | Stewardship | Private review, outcome documented |

---

## Working Group Formation

To propose a new working group:
1. Open an issue using the **Working Group Proposal** template
2. Demonstrate interest: at least 3 people willing to participate
3. Define the scope clearly enough to avoid overlap with existing groups
4. Stewardship reviews and approves within 14 days
