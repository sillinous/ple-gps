# APQC PCF — Process Maturity Model

Based on APQC's Process and Enterprise Maturity Model (PEMM) and
Carnegie Mellon's CMMI, adapted for PCF-aligned assessment.

---

## Five Maturity Levels

| Level | Name | Description | Key Characteristic |
|-------|------|-------------|-------------------|
| 1 | Ad Hoc | Processes are informal, undocumented, and person-dependent | Heroics over systems |
| 2 | Defined | Processes are documented and consistently followed | Repeatability |
| 3 | Standardized | Processes are optimized, measured, and aligned | Efficiency |
| 4 | Managed | Processes are quantitatively controlled | Predictability |
| 5 | Optimizing | Processes continuously improve using data and AI | Self-improving |

---

## Maturity Dimension Assessment (5 Dimensions per Process)

For each L2 Process Group, assess across five dimensions:

### Dimension 1: Design
| Level | Description |
|-------|-------------|
| 1 | Process undocumented; steps vary by person |
| 2 | Process documented in basic SOP/workflow |
| 3 | Process designed to industry best practice; peer-reviewed |
| 4 | Process designed using data and simulation |
| 5 | Process continuously redesigned based on performance feedback |

### Dimension 2: Performers
| Level | Description |
|-------|-------------|
| 1 | Staff improvise; no standard knowledge required |
| 2 | Staff have basic training; can follow documented SOP |
| 3 | Staff are specialized and certified; performance is measured |
| 4 | Staff use decision support tools; output is predictable |
| 5 | AI augments staff; automation handles routine execution |

### Dimension 3: Owner
| Level | Description |
|-------|-------------|
| 1 | No process owner exists |
| 2 | Process owner identified but role is informal |
| 3 | Process owner has formal authority and accountability |
| 4 | Process owner manages with KPIs and governance cadence |
| 5 | Process owner drives continuous improvement using AI insights |

### Dimension 4: Infrastructure
| Level | Description |
|-------|-------------|
| 1 | No system support; fully manual |
| 2 | Basic tools used (spreadsheets, email) |
| 3 | Dedicated systems deployed (ERP, CRM, etc.) |
| 4 | Systems are integrated; data flows automatically between processes |
| 5 | AI/ML embedded in systems; predictive and prescriptive capabilities |

### Dimension 5: Metrics
| Level | Description |
|-------|-------------|
| 1 | No metrics tracked |
| 2 | Basic output metrics tracked periodically |
| 3 | Balanced set of KPIs tracked with targets; reviewed regularly |
| 4 | Predictive metrics and leading indicators used |
| 5 | Real-time metrics feed AI models that auto-optimize the process |

## Improvement Roadmap by Maturity Band

### Moving 1→2 (Ad Hoc → Defined)
**Priority:** Document and standardize
- Map the current "as-is" process
- Identify the steps actually happening
- Write the basic SOP
- Assign a process owner (even if informal)
- Train performers
- Start tracking one basic output metric

### Moving 2→3 (Defined → Standardized)
**Priority:** Optimize and align
- Benchmark against APQC best practice
- Redesign process to eliminate waste (Lean)
- Formalize the process owner role (authority + accountability)
- Deploy dedicated tools/systems
- Implement balanced KPI scorecard
- Establish regular process review cadence

### Moving 3→4 (Standardized → Managed)
**Priority:** Instrument and predict
- Instrument process with real-time data collection
- Build predictive models for key process outputs
- Integrate this process with adjacent processes (fix integration gaps)
- Automate routine decisions using rules engines
- Implement continuous monitoring with alerting
- Use statistical process control (SPC) where applicable

### Moving 4→5 (Managed → Optimizing)
**Priority:** Automate and learn
- Embed AI/ML into process execution
- Automate all routine steps end-to-end
- Build feedback loops that continuously improve the process
- Use AI to identify improvement opportunities proactively
- Shift human role to exception handling and strategy
- Create self-healing process design

## Strategic Maturity Targeting

**Not all processes should be at Level 5.** Target maturity should reflect:

1. **Strategic differentiation value** — Processes that create competitive advantage
   should target Level 4–5
2. **Volume and cost impact** — High-volume processes with large cost should target
   Level 3–4 minimum for efficiency
3. **Risk exposure** — Risk/compliance processes should target Level 4 for predictability
4. **Customer impact** — Customer-facing processes should target Level 4–5 for experience quality

### Recommended Maturity Targets by Category

| Category | Recommended Target | Rationale |
|----------|-------------------|-----------|
| 1.0 Strategy | Level 3 | Needs human judgment; over-automating kills adaptability |
| 2.0 Products | Level 3–4 | Innovation needs flexibility; quality needs control |
| 3.0 Market & Sell | Level 4 | High ROI from CRM, marketing automation, AI pipeline mgmt |
| 4.0 Deliver Physical | Level 4–5 | Volume + cost = high automation ROI |
| 5.0 Deliver Services | Level 3–4 | Service needs flexibility; delivery management needs control |
| 6.0 Customer Service | Level 4–5 | High volume + experience impact = AI priority |
| 7.0 Human Capital | Level 3 | People processes need human judgment at core |
| 8.0 IT | Level 4 | IT enables others; must be reliable and integrated |
| 9.0 Finance | Level 4–5 | High-volume transaction processing → automation |
| 10.0 Assets | Level 3–4 | Predictive maintenance ROI is strong |
| 11.0 Risk | Level 4 | Continuous monitoring critical; can't be periodic |
| 12.0 External | Level 3 | Relationship-intensive; human judgment essential |
| 13.0 Capabilities | Level 3–4 | Meta-processes need discipline but also adaptability |

## APQC Maturity Benchmarks

Based on APQC research across 2,000+ organizations:

| Maturity Level | % of Organizations |
|---------------|-------------------|
| Level 1 (Ad Hoc) | 15% |
| Level 2 (Defined) | 40% |
| Level 3 (Standardized) | 30% |
| Level 4 (Managed) | 12% |
| Level 5 (Optimizing) | 3% |

Most organizations are at Level 2 across the majority of their processes.
Moving from Level 2→3 is the highest-ROI improvement for most organizations.
