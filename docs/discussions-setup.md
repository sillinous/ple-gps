# GitHub Discussions Setup Guide

GitHub Discussions categories must be created through the GitHub web UI (Settings > Discussions).
This file documents the intended category structure so you can set it up consistently.

---

## Categories to Create

### 📣 Announcements
- **Type:** Announcements (only maintainers can post, anyone can reply)
- **Description:** Project updates, releases, community news, and events
- **Format:** Discussion

### 👋 Introductions
- **Type:** Open (anyone can post)
- **Description:** New to the community? Introduce yourself here.
- **Format:** Discussion

### 💭 Theory & Frameworks
- **Type:** Open
- **Description:** Open-ended exploration of PLE concepts, economic models, and theoretical questions
- **Format:** Discussion

### 🛠 Technical
- **Type:** Open
- **Description:** Platform development, tools, infrastructure, and technical questions
- **Format:** Discussion + Q&A

### 📋 Working Groups
- **Type:** Open
- **Description:** Per-working-group update threads and coordination
- **Format:** Discussion
- **Note:** Create a pinned thread for each WG: WG-THEORY, WG-POLICY, WG-TECH, WG-COMMUNITY, WG-COMMS, WG-GOVERNANCE

### 🌍 Regional
- **Type:** Open
- **Description:** Local organizing, regional implementation, and community-specific discussions
- **Format:** Discussion

### 🤔 Open Questions
- **Type:** Q&A
- **Description:** Things we don't know yet — questions looking for answers
- **Format:** Q&A

### 📜 Decisions Log
- **Type:** Announcements (maintainers post decisions, community can comment)
- **Description:** Archive of Type 2/3/4 decisions with rationale
- **Format:** Discussion
